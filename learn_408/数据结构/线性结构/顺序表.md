# 顺序表

[[toc]]

## 基本概念

顺序表是线性表的顺序存储实现，其特点在于用一段连续相邻的地址存储元素，使得逻辑上相邻的元素物理上也相邻

## 优缺点

### 空间性能上

- 存储空间的分配上：顺序表的存储空间必须预先分配，元素个数扩充受一定限制，易造成存储空间浪费或空间溢出现象；
- 存储密度的大小上：顺序表不需要用额外的空间来表示元素之间的关系，其存储空间完全用于存储数据元素，所以存储密度高

### 时间性能上

- 存取元素的效率上：顺序表是由数组实现的，它是一种随机存取结构，指定任意一个位置序号 i 都可以在 `O(1)`时间内直接存取该位置上的元素， 即取值操作的效率高；
- 插入和删除操作的效率上：对于顺序表，进行插入或删除时，平均要移动表中近一半的结点，时间复杂度为`O(n)`。

## 基本实现

::: warning 注意

1. js 采用数组作为底层容器，但是 js 数组会有空洞的问题。在遍历时要注意遍历算法的选择，以及扩容和缩容的策略。
2. 长度和容量是不一样的。长度指的是实际存储的元素个数，容量指的是数组的总长度。
3. 顺序表的核心在于逻辑上相邻的元素物理上也相邻，所以在插入和删除元素时，需要移动元素，让元素在物理位置上也是相邻的

:::

### 基本数据结构

区分以下概念：

- 顺序表的长度：指的是顺序表中实际存储的元素个数；
- 顺序表的容量：指的是顺序表的最大容量，即顺序表的存储空间大小；

```js
/**
 * 顺序表（SeqList）实现
 * @template T 顺序表中元素的类型
 */
export class SeqList<T> {
  private elements: T[];
  private length: number;
  private capacity: number;
  /**
   * 构造函数
   * @param initialCapacity 初始容量，默认为10
   */
  constructor(initialCapacity: number = 10) {
    this.elements = new Array(initialCapacity);
    this.capacity = initialCapacity;
    this.length = 0;
  }
  /**
   * 获取顺序表当前长度
   * @returns 顺序表当前元素数量
   */
  public getLength(): number {
    return this.length;
  }

  /**
   * 检查顺序表是否为空
   * @returns 如果为空返回true，否则返回false
   */
  public isEmpty(): boolean {
    return this.length === 0;
  }

  /**
   * 在顺序表末尾添加元素
   * @param element 要添加的元素
   */
  public append(element: T): void {
    // 判断是否需要扩容
    if (this.length >= this.capacity) {
      this.resize(this.capacity * 2);
    }
    // 在数组末尾添加元素
    this.elements[this.length] = element;
    // 长度加1
    this.length++;
  }

  /**
   * 获取指定位置的元素
   * @param index 元素位置
   * @returns 指定位置的元素
   * @throws 如果索引越界则抛出错误
   */
  public get(index: number): T {
    // 判读索引是否越界
    if (index < 0 || index >= this.length) {
      throw new Error('Index out of bounds');
    }
    return this.elements[index];
  }

  /**
   * 设置指定位置的元素
   * @param index 元素位置
   * @param element 新元素
   * @throws 如果索引越界则抛出错误
   */
  public set(index: number, element: T): void {
    // 判断索引是否越界
    if (index < 0 || index >= this.length) {
      throw new Error('Index out of bounds');
    }
    // 设置元素
    this.elements[index] = element;
  }

  /**
   * 查找元素首次出现的位置
   * @param element 要查找的元素
   * @returns 元素首次出现的索引，如果未找到则返回-1
   */
  public indexOf(element: T): number {
    return this.elements.indexOf(element);
  }

  /**
   * 检查顺序表是否包含指定元素
   * @param element 要检查的元素
   * @returns 如果包含则返回true，否则返回false
   */
  public contains(element: T): boolean {
    return this.elements.indexOf(element) !== -1;
  }

  /**
   * 清空顺序表
   */
  public clear(): void {
    // 重置数组原来数组会被垃圾回收
    this.elements = new Array(0);
    this.capacity = 10;
    this.length = 0;
  }

  /**
   * 将顺序表转换为数组
   * @returns 包含所有元素的数组
   */
  public toArray(): T[] {
    return this.elements;
  }

}

```

### 容量变化方法

::: tip 垃圾回收机制
旧数组空间没有引用指向时，会被垃圾回收机制回收
:::

```js
 /**
   * 调整顺序表容量
   * @param newCapacity 新的容量
   */
  private resize(newCapacity: number): void {
    // 创建以新的容量为大小的新数组
    const newElements = new Array(newCapacity);
    // 复制原数组元素到新数组
    for (let i = 0; i < this.length; i++) {
      newElements[i] = this.elements[i];
    }
    // 更新顺序表的元素数组和容量
    this.elements = newElements;
    this.capacity = newCapacity;
  }
```

### 在指定位置上插入元素

插入元素的三步：

- 插入位置越界判断
- 容器扩容
- 移动元素

```js
  /**
   * 在指定位置插入元素
   * @param index 插入位置
   * @param element 要插入的元素
   * @throws 如果索引越界则抛出错误
   */
  public insert(index: number, element: T): void {
    // 判断索引是否越界
    if (index < 0 || index > this.length) {
      throw new Error('Index out of bounds');
    }
    // 检测是否需要扩容
    if (this.length >= this.capacity) {
      this.resize(this.capacity * 2);
    }
    // 在数组指定位置插入元素
    for (let i = this.length; i > index; i--) {
      this.elements[i] = this.elements[i - 1];
    }
    // 插入新元素
    this.elements[index] = element;
    // 长度加1
    this.length++;
  }
```

### 在指定位置上删除元素

删除元素的四步：

- 长度为空判断
- 删除位置越界判断
- 移动元素
- 容器缩容：注意容器缩容时，元素个数为容积的 1/4 时才缩容，容量是整数，要避免为 0

```js
 /**
   * 删除指定位置的元素
   * @param index 要删除的元素位置
   * @returns 被删除的元素
   * @throws 如果索引越界或顺序表为空则抛出错误
   */
  public remove(index: number): T {
    // 判断顺序表是否为空
    if (this.isEmpty()) {
      throw new Error('SeqList is empty');
    }
    // 判断索引是否越界
    if (index < 0 || index >= this.length) {
      throw new Error('Index out of bounds');
    }
    // 获取被删除的元素
    const element = this.elements[index];
    // 遍历数组，将元素前移
    for (let i = index; i < this.length - 1; i++) {
      this.elements[i] = this.elements[i + 1];
    }
    // 长度减1
    this.length--;
    // 缩短容量（当元素个数为容积的1/4时缩短容量 ，容量是整数，要避免为0）
    if (this.length <= this.capacity / 4 && this.capacity / 2 > 0) {
      this.resize(this.capacity / 2);
    }
    // 返回删除的元素
    return element;
  }
```

### 遍历顺序表

::: tip 数组元素的空洞
有些数组方法不会跳过空洞要注意，具体看 JS 数组部分
:::

```js
 /**
   * 遍历顺序表中的所有元素
   * @param callback 对每个元素执行的回调函数
   */
  public forEach(callback: (element: T, index: number) => void): void {
    for (let i = 0; i < this.length; i++) {
      callback(this.elements[i], i); // 执行回调函数
    }
  }
```

## 额外操作实现

### 加入迭代器实现顺序表可迭代

::: tip 迭代器
生成器对象即使迭代器对象也是可迭代对象
:::

```js
  // 正向迭代器
  *[Symbol.iterator](): IterableIterator<T> {
    for (let element of this.elements) {
      if (element !== undefined) {
        yield element;
      }
    }
  }
  // 反向遍历迭代器
  *reverseIterator(): IterableIterator<T> {
    for (let i = this.length - 1; i >= 0; i--) {
      yield this.elements[i];
    }
  }

```

使用

```js
console.log('正向遍历:');
for (const num of numberList) {
  console.log(num);
}
console.log('反向遍历');
for (const num of numberList.reverseIterator()) {
  console.log(num);
}
```
